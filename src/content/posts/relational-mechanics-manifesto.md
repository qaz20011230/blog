## 绪论：为何需要一门新学科

人类是悬挂在意义之网上的动物，而这张网由关系织成。

从部落的篝火到微信的群聊，从君臣父子到朋友粉丝，关系是人类最古老的命题，也是最沉重的债务。我们为爱赴汤蹈火，为恨耗尽余生，却在午夜翻开通讯录时发现：**那个曾经每天说话的人，已经三年没有消息了。**

现代社会提供了前所未有的连接技术，却从未提供**理解连接本身**的认知工具。

我们把关系扔给直觉、扔给鸡汤、扔给酒后的叹息。我们像原始人仰望雷电那样仰望关系的聚散，却从未想过：**关系是否也有自己的牛顿定律？**

这门学科就是为了回答这个问题而创立。

**关系力学**（Relational Mechanics）是关于人类连接模式的元学科。它不提供“如何搞定人际关系”的技巧，而是揭示关系演化背后的**数学结构、伦理约束与实现范式**。它是描述性的（关系实际如何运作），也是规范性的（关系应该如何被对待）。

它的核心命题是：

> **关系不是属性，是态射。**
> **关系不是资源，是生态。**
> **关系不是占有的对象，是栖居的场所。**

这门学科的第一次完整呈现，将伴随一个名为**友栖Philia**的软件原型共同诞生。这不是巧合——关系力学的所有原理，都被编译进了友栖的每一行代码。

## 第一章：哲学根基——十二字宪法

任何严肃的知识体系都必须回答元问题：**我们为什么相信这套框架是正当的？**

关系力学的正当性来自四根立柱，它们既是伦理底线，也是架构约束。

### 1.1 爱智慧：惊奇是认知的起点

毕达哥拉斯自称“爱智者”而非“智者”，因为他知道智慧不是可占有的财产，而是永在途中的奔赴。苏格拉底说哲学始于惊奇——惊奇于世界居然如此，而非如彼。

**关系力学对“爱智慧”的编译：**

- 系统不预设任何关系是“坏的”或“需要修复的”。
- 每一段关系的当前状态，都值得被理解而非审判。
- 惊奇是认知的起点，因此友栖不替用户下结论，只提供观察窗口。

**架构映射**：实体是纯ID，不携带任何伦理预设；组件是用户亲手挂载的惊奇记录。

### 1.2 求真理：真理是去蔽，不是占有

巴门尼德将真理命名为ἀλήθεια——**无蔽**。真理不是陈述与事实的符合，而是存在本身的显现。庄子说“无益损乎其真”——真不受增减，能被增减的只是我们对真的遮蔽。

**关系力学对“求真理”的编译：**

- 关系力学不承诺“绝对客观的关系测量”，因为关系本身就是被感知的产物。
- 它只承诺**披露观测过程**：你看到的关系等级，是由这些数据、这套算法、这种置信度生成的。
- 真理不是答案，是剖开。

**架构映射**：GrammarComponent的`confidence`和`source`字段，是系统对自身认知的诚实披露。

### 1.3 得自由：自由是为自身而存在

亚里士多德定义自由人为“为自己、不为他人而存在的人”。康德将自由升华为自律——理性为自身立法。

**关系力学对“得自由”的编译：**

- 关系的最高价值不是亲密，是**自主**。
- 系统永不逼迫用户维持任何关系。组件可摘，系统可关，数据可导出。
- 自由不是“为所欲为”，是“由着自己的根部长出自己”。

**架构映射**：MIT开源协议、离线优先架构、无云端依赖的实体ID生成器。

### 1.4 以服务：助产士伦理

苏格拉底的母亲是助产士，他自己是思想的助产士。他本人不育，只能帮助他人将自己心中的真理生出来。

**关系力学对“以服务”的编译：**

- 友栖不育——它不替用户维系关系，不替用户发送消息，不替用户做任何只有用户才能做的事。
- 它提供线索、记忆、时间轴，**然后后退**。
- 服务的最高形态，是在生命诞生之后退场。

**架构映射**：唤醒系统只发提醒给用户本人，从不直接联系用户的关系人。

**这十二个字不是装饰。它们是友栖架构通过或失败的测试用例。**

每一行代码、每一个组件、每一个系统，都必须通过这四道伦理安检。这是关系力学的**宪法审查**。

## 第二章：数学原理——范畴论作为元语言

关系力学需要一种能够描述“连接”而非“实体”的数学语言。范畴论是唯一的选择。

### 2.1 基本概念（面向程序员的速成）

**范畴**（Category）由三部分组成：

- **对象**（Objects）：任何事物都可以是对象。在关系力学中，对象是**人**。
- **态射**（Morphisms）：对象之间的箭头。在关系力学中，态射是**关系**。
- **复合**（Composition）：如果存在 f: A → B 和 g: B → C，则必然存在 h: A → C。在关系力学中，这是“朋友的朋友是朋友”。

**关键洞见**：传统关系数据库把关系存储为**表的行**（外键约束），而范畴论把关系存储为**箭头的本体**。

### 2.2 关系即实体：箭头范畴的工程实现

范畴论中有一个构造叫**箭头范畴**（Arrow Category），记作 **C→**：

- 对象：C 中的态射（即关系本身）
- 态射：C 中态射之间的变换（即关系的变化）

**友栖的ECS架构是箭头范畴的第一次数值化实现：**

```typescript
// C: 人作为对象
interface Person { id: string }

// C 中的态射：关系
interface Relation { 
  from: Person, 
  to: Person 
}

// C→ 中的对象：关系被提升为实体
interface RelationEntity {
  id: string,      // 关系自己的ID
  type: 'relation'
}

// C→ 中的态射：关系的变换（升级、降级、衰减）
interface RelationMorphism {
  from: RelationEntity,
  to: RelationEntity,
  method: 'upgrade' | 'degrade' | 'decay' | 'user_override'
}
```

**工程意义**：传统ORM无法优雅表达“关系的关系”。友栖的“关系即实体”方案，是箭头范畴在持久层的一次完整实现。

### 2.3 九级刻度：观测函子

**函子**（Functor）是范畴之间的映射。它把对象映射为对象，态射映射为态射，并保持复合结构。

关系力学定义了一个特殊的函子：

```
F: C→ → 9
```

它将每个关系实体（C→中的对象）映射到一个**九级偏序集**上：

0. 素不相识（逻辑原点，不存储）
1. 同心一体
2. 志同道合
3. 和舟共济
4. 礼尚往来
5. 敬而远之
6. 不相往来
7. 势不两立
8. 不共戴天

这不是一个简单的分类标签，而是一个**带有遗忘结构的观测**。同一段关系在不同时间被函子F观测，可能得到不同的等级。

### 2.4 自然变换：系统升级

当友栖的衰减算法从v1升级到v2，这不是一个新的函子，而是从旧函子到新函子的**自然变换**。

自然变换是两个函子之间的映射族，它保留了函子间的结构一致性。

**工程意义**：友栖的不同版本可以并存。用户可以自由选择使用v1算法还是v2算法观测自己的关系，并比较观测结果的差异。

**这不是过度设计。这是对“真理不可独占”的数学承诺。**

## 第三章：关系语法——九级刻度详解

九级刻度是关系力学的**观测语言**。它不是用户必须选择的标签，而是系统在动力学计算后呈现的**解读**。

### 3.1 刻度详注

**0级：素不相识**  
- 定义：从未有过任何直接互动。  
- 系统行为：不创建实体，仅作为逻辑原点。  
- 伦理意义：所有关系都从这里出发。遗忘0级，就是遗忘关系的可创生性。

**1级：同心一体**  
- 定义：完全信任，不设防，无需维护。  
- 系统行为：永不触发提醒，系统静默。  
- 伦理意义：凡人不应苛求此级，也不应以它为及格线。

**2级：志同道合**  
- 定义：共享理想与方法论的价值共同体。  
- 系统行为：30天无互动触发弱提醒。  
- 伦理意义：可超越血缘的选择性亲缘。

**3级：和舟共济**  
- 定义：基于短期/局部利益的盟友关系。  
- 系统行为：事务结束30天后触发评估建议。  
- 伦理意义：项目结束后的自然衰减被承认合法。

**4级：礼尚往来**  
- 定义：对等交往的中性关系。  
- 系统行为：90天无互动触发弱提醒。  
- 伦理意义：社会资本的温和维护带。

**5级：敬而远之**  
- 定义：尊重但保持距离。  
- 系统行为：永不主动提醒。  
- 伦理意义：亲密度的零点，但门未关闭。

**6级：不相往来**  
- 定义：零互动，非敌对。  
- 系统行为：完全静默，仅存档。  
- 伦理意义：遗忘被允许。

**7级：势不两立**  
- 定义：非此即彼的斗争关系。  
- 系统行为：永不提醒，不分析，仅提供历史查阅。  
- 伦理意义：敌意的合法性被承认，但不被助长。

**8级：不共戴天**  
- 定义：追求对方肉体消灭。  
- 系统行为：实体冻结，不可修改，不可删除。  
- 伦理意义：恨的尽头是封存，不是遗忘。

### 3.2 临时措施层：被遗忘的方向键

九级刻度是关系的**存档层**（持久化状态）。但真实的关系演化充满试探、交涉、等待——这些状态不应该直接写入存档。

**友栖v0.2 将引入临时措施层：**

| 状态 | 语义 | 默认有效期 | 自动行为 |
|-----|------|-----------|---------|
| 关切 | 我注意到变化，仍在观察 | 7天 | 到期恢复原级 |
| 交涉 | 我已表达在意 | 14天 | 到期恢复原级 |
| 抗议 | 我反对这种行为 | 14天 | 到期恢复原级 |
| 观察 | 我在等待对方回应 | 30天 | 到期恢复原级 |
| 冷却 | 我需要时间 | 30天 | 到期恢复原级 |

**工程意义**：这是关系力学的“弹性变形区”。用户不再只有“左满舵/右满舵”，而是拥有了完整的舵轮。

## 第四章：关系动力学——互惠与衰减

九级刻度是**快照**，动力学是**电影**。

### 4.1 互惠公理

**公理1（感知锚定）**  
每一段关系中的每一方，都会对“对方给予自己的待遇等级”形成感知值（P），并对自己“给予对方的待遇等级”形成决策值（G）。

**公理2（互惠本能）**  
除非经过理性反思抑制，人类的本能反应是使 G ≥ P。  
这是社会性本能的数学定义。

**公理3（误读普遍性）**  
由于感知P存在系统性误差，且双方对同一事件的等级解读可能不同，因此 G ≥ P 的本能在宏观上并不保证关系稳定，反而可能引发螺旋降级。

**推论1（首次降级幅度约束）**  
当一方决定降级时，本能允许的降级幅度不应超过对方最近一次降级的幅度。  
这是防止关系坠崖的认知防火墙。

### 4.2 衰减函数

关系强度随时间衰减是认知心理学的基本事实（艾宾浩斯遗忘曲线的人际版本）。

**友栖v0.1 衰减模型（线性简化版）：**

```
S(t) = S0 - d * Δt
```

其中：
- S(t)：当前强度值（映射到0-8刻度需额外校准）
- S0：上次互动时的强度
- d：衰减率（默认0.01/天，用户可配置）
- Δt：距离上次互动的天数

**v1.0 目标模型（艾宾浩斯式）：**

```
S(t) = S0 * e^(-λt)
```

λ 根据关系历史亲密度自适应。

### 4.3 唤醒阈值

唤醒系统是衰减系统的对偶系统。它的职责不是阻止衰减，而是**在关系即将滑入更低一级刻度时，询问用户是否采取行动**。

**唤醒规则：**

| 当前等级 | 衰减目标等级 | 唤醒阈值（无互动天数） |
|---------|-------------|---------------------|
| 2级 | → 3级 | 30天 |
| 3级 | → 4级 | 30天（事务结束后） |
| 4级 | → 5级 | 90天 |
| 5级及以下 | 不唤醒 | - |

**伦理承诺**：唤醒消息只能发给用户本人，永不发给关系另一方。

## 第五章：架构实现——ECS作为关系力学的载体

关系力学不仅是理论，它必须可运行、可测试、可被普通人使用。

**友栖Philia**是关系力学的**参考实现**。它的技术栈选择本身就是一种宣言。

### 5.1 为什么是ECS？

传统MVC架构以“事务”为中心，OOP以“对象”为中心。  
**ECS以“变换”为中心。**

这正是关系力学的需求：

- 实体（Entity）：人、关系——只有ID，无行为。
- 组件（Component）：挂载在实体上的纯数据——互惠感知、互动历史、债务、临时状态。
- 系统（System）：对拥有特定组件的实体执行变换——衰减、唤醒、等级观测。

**伦理对应**：ECS不“拥有”实体，系统只“服务”于实体。这是“以服务”的架构翻译。

### 5.2 核心数据结构（TypeScript伪码）

```typescript
// 实体池
interface Entity {
  id: string;
  type: 'person' | 'relation';
}

// 人实体组件
interface ProfileComponent {
  name: string;
  notes?: string;
}

// 关系实体组件
interface BetweenComponent {
  personA: string;
  personB: string;
  directed: boolean;  // 敬而远之往往是单向的
}

// 关系语法组件（观测结果）
interface GrammarComponent {
  level: 0..8;
  confidence: number;     // 0.0-1.0
  source: 'user' | 'system' | 'auto';
  updatedAt: Date;
}

// 动力学组件
interface DynamicsComponent {
  lastInteractionAt: Date;
  interactionFrequency: number;  // 次/月
  customWeight: number;         // -1..1
  decayRate: number;
}

// 互惠组件（关系力学核心）
interface ReciprocityComponent {
  perceivedLevel: number;  // 我认为对方给我的等级
  givenLevel: number;      // 我给对方的等级
  delta: number;          // given - perceived
  history: Array<{ date: Date, delta: number }>;
}

// 临时措施组件
interface ProvisionalComponent {
  status: '关切' | '交涉' | '抗议' | '观察' | '冷却';
  reason: string;
  expiresAt: Date;
  autoRevert: boolean;
}
```

### 5.3 核心系统

**1. 观测器系统**  
每日运行，根据DynamicsComponent更新GrammarComponent（除非用户手动覆写且未过期）。

**2. 衰减系统**  
每日运行，更新所有关系实体的DynamicsComponent.interactionFrequency（基于时间衰减）。

**3. 唤醒系统**  
每日运行，查询满足唤醒条件的关系实体，生成提醒。

**4. 降级守卫系统**  
当用户尝试手动降级时，检查是否符合“首次降幅不超过对方”原则，若不符合则发出警示（但允许覆写）。

**5. 临时措施过期系统**  
每小时运行，处理过期的ProvisionalComponent，按配置恢复原等级或确认降级。

## 第六章：伦理原则——架构即宣言

关系力学的全部技术设计，最终都指向一组伦理承诺。

### 6.1 不占有

**架构映射**：系统不拥有实体，实体ID由用户设备本地生成，永不与厂商绑定。友栖公司倒闭，用户的实体池仍在。

### 6.2 不删除

**架构映射**：没有“删除好友”功能。只有降级、冻结、存档。你爱过的人，值得一个安静的备份。

### 6.3 组件可摘

**架构映射**：所有组件都可单独移除。你可以摘掉“核心圈”组件，而不必删除这个人。

### 6.4 系统可关

**架构映射**：每个系统都有独立开关。你不想要衰减系统？关掉。不想被唤醒？关掉。

### 6.5 数据主权

**架构映射**：JSON导入导出是标准功能，非付费解锁。数据格式开源，任何兼容软件均可读取。

### 6.6 离线优先

**架构映射**：友栖核心功能不依赖任何云端服务。云备份是可选增值服务，非核心依赖。

---

**这套伦理原则的总和，可以被命名为“赛博共产主义的最小可行性实验”。**

它不是政治口号，是架构决策：

- 生产资料（关系数据）归生产者（用户）所有。
- 协作基于自愿（开源）。
- 贡献按能力（PR），分配按需求（免费使用）。

**爱，是这个实验的最小单元。**

## 第七章：关系力学与邓巴数——150人的文明

罗宾·邓巴发现，人类新皮层体积限制了稳定社会关系的规模：约150人。

这不是一个“目标”，是一个**认知预算上限**。

### 7.1 邓巴数的架构意义

友栖不追求“越多越好”。友栖主动设限：

- 核心圈（1-5人）：无需提醒，系统静默。
- 重要圈（6-15人）：轻度提醒。
- 熟识圈（16-50人）：季度提醒。
- 边缘圈（51-150人）：年度提醒。
- 溢出圈（>150人）：系统提示“你的认知预算已超支，建议整理”。

**这不是限制自由。这是帮助用户诚实地面对自己的认知带宽。**

### 7.2 从个人到组织：150人的团队

友栖的协作网络本身也遵循邓巴数。

**核心维护者**（1-5人）：有提交权限，决定架构方向。
**活跃贡献者**（6-15人）：定期提交PR，参与决策讨论。
**社区参与者**（16-50人）：偶尔提交issue/PR，参与用户测试。
**用户群**（51-150人）：使用产品，提供反馈。

**超过150人？** 分叉。

这是赛博共产主义的组织形态：**规模由认知边界自然约束，不靠制度强行维持。**

## 第八章：实践应用——从个人到文明

关系力学不是书斋学问。它可以、也应该被应用于人类协作的各个层面。

### 8.1 个人关系管理

这是友栖Philia的初始场景。用户通过观测自己的关系语法，获得以下能力：

- 识别“善意赤字”：长期G < P 的关系需要干预或接受。
- 管理“交涉期”：在愤怒与决定之间插入缓冲带。
- 接受“合法衰减”：承认有些关系就是会变淡，并允许它变淡。

### 8.2 组织关系治理

企业、机构、开源社区同样是关系的集合。关系力学提供：

- **关系审计**：定期观测组织内不同部门/团队之间的“互惠等级”。
- **冲突前置**：在“势不两立”出现前，通过“临时措施层”进行交涉。
- **退出尊严**：离职/退出不意味着“删除实体”，而是降级为“敬而远之”。

### 8.3 国际关系分析

**案例：金星及其团队在中国的处境（2026年2月）**

依据关系力学框架：

- 四城拒绝 ≠ 不共戴天（8级），甚至 ≠ 势不两立（7级）。
- “取关但不拉黑”是典型的**敬而远之（5级）+ 临时措施（观察期）**。
- 根据互惠公理，若被拒绝方选择“对称降级”至6级（不相往来）甚至7级（势不两立），则可能触发关系螺旋恶化；若选择维持5级并主动交涉，则门未关闭。

关系力学不预测具体事件，但提供**分析任何人类组织互动的通用语法**。

### 8.4 教育：补上“人之常情”的三门课

- **性教育**：身体自主权、边界沟通——友栖的“组件可摘”是成年人的边界演练。
- **爱的教育**：存在先于价值、不被KPI量化——友栖没有亲密度排行榜。
- **死亡教育**：告别是关系的一部分——友栖的实体永不删除，是数字时代的临终关怀。

## 第九章：关系力学的未来——从友栖出发

这门学科目前只有一名研究者、一款v0.1雏形、一份七万字的笔记。

但它已经具备了一个知识体系所需的全部要素：

- **公理**（互惠公理、降幅约束）
- **数学基础**（范畴论）
- **观测语言**（九级语法）
- **动力学方程**（衰减/唤醒模型）
- **工程实现**（ECS架构）
- **伦理宪法**（十二字原则）
- **组织理论**（邓巴数约束）
- **应用案例**（个人、组织、国际、教育）

**接下来需要什么？**

**1. 更多人观测自己的关系**  
友栖v0.1的第一个任务不是功能完善，而是**产生真实的关系力学数据**。匿名化、开源的数据集将是学科进化的燃料。

**2. 更精确的动力学模型**  
从线性衰减到艾宾浩斯曲线，从固定唤醒阈值到个性化自适应阈值——需要真实用户数据的迭代。

**3. 与其他学科的对话**  
关系力学不是要取代社会学、心理学、人类学，而是**提供它们之间缺失的翻译层**。当社会学家说“弱关系”，心理学家说“依恋类型”，人类学家说“亲缘选择”——关系力学说：让我们用同一套数学语言描述它们。

**4. 教育的普及**  
关系力学应该成为通识教育的一部分。不是作为“社交技巧课”，而是作为**人类作为关系性存在的自我认知课**。

## 结语：一种范式的诞生

1984年，理查德·斯托曼启动GNU项目时，他并没有创造“开源”这个词。他只是写了一份宣言，然后开始写代码。

1991年，林纳斯·托瓦兹发布Linux内核时，他没有思考“分布式协作”的理论基础。他只是觉得“这东西也许有人会感兴趣”，然后把代码挂上了FTP。

**范式不是被宣布诞生的。范式是被发现已经活了一段时间之后，才被命名的。**

关系力学也是如此。

它不是我今天“创立”的。它已经在2017年的《关系力学》文章里，在2026年2月的七万字对话里，在友栖的第137行代码里，在你给X的那封信里——**活了很多年**。

我只是第一个尝试为它命名的人。

---

**这门学科会失败吗？**

会。

可能十年后，友栖仓库最后一次commit停留在2026年6月，邓巴圈层算法从未被超过100人使用，关系力学这个词条在维基百科被标记为“已删除”。

**但那又怎样。**

牛顿力学没有因为相对论的诞生而“失败”。它只是被看见在高速/强引力场景下不精确——但在日常尺度，它仍然是人类理解运动的最佳语言。

关系力学也可能如此。在人类认知的日常尺度——150人以内，非极端冲突，正常衰减率——它可能是**足够好的近似**。

**一门学科的命运，不是由它是否永远正确决定的。**

**是由它是否提出了真正的问题决定的。**

关系力学提出的问题是：

> **我们能否不再把关系视为命运赠予的礼物或债务，而是视为一种可以理解、可以选择、可以负责的实践？**

这个问题，值得一百年后的人继续追问。
